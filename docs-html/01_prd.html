1要確認

  1. 現状の実測値: 新規アプリ1件の公開環境準備にかかる時間、月次のインフラ保守時間、脆弱性対応の年間発生回数

  3. 予算20万円の使途の細目(Azure利用料のみか、ツール・ライセンス費を含むか)
  4. 本番展開・既存アプリ移行を判断する時期

2目的

社内研究テーマ「アイデア探求ワーク」の一環として、AI駆動開発で構築したアプリをAzure上で短時間で公開できる基盤を構築する。基盤は次の2つを作り、今後のアプリ公開のテンプレートとして提示できる状態にする。

静的コンテンツ公開用の基盤

ビルド済みの静的サイトを配信する

動的コンテンツ公開用の基盤

Python・Next.jsのアプリをデプロイする

3ゴール(検証の完了条件)

  • G-1静的基盤: テンプレートから静的サイトを1本公開できる
  • G-2動的基盤: PythonとNext.jsのアプリを各1本公開できる
  • G-32基盤それぞれで、新規公開1件の所要時間と月額費用を実測し、現行VM運用と比較した結果を提示できる
  • G-4検証結果をもとに、本番展開の可否を判断できる材料が揃っている

4対象業務と利用部門

チーム内のアプリ公開・運用業務。検証フェーズのため、利用は起案者のチームに限定する。

5現状の課題

Azure VM(Ubuntu)にDockerを載せ、3種のコンポーネントでアプリを公開している。

Azure VMでの運用は、構成を自由に組める柔軟性と月額費用の低さで優れている。一方でIaaS運用のため次の2点が負担であり、この負担は基盤の数に比例して増える。

  • 新規公開のたびにVM上での初期構築作業が発生し、公開までのリードタイムが伸びる
  • OS・ミドルウェアの脆弱性対応を自チームで担っており、対応が遅れた期間はセキュリティリスクを抱える
