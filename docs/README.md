# docs — 案件書類

実際の案件の書類をここに置く。雛形は [docs-template/](../docs-template/) にある。

## 使い方

1. 案件開始時、必要な雛形を `docs-template/project/` からこの `docs/project/` にコピーする
2. 雛形内の記入例(`<details>` ブロック)と記入ガイドの引用部分を削除し、案件の内容を記入する
3. 変更は PR で行い、マージを承認記録とする

## 構成

- `project/` — 案件書類(企画書〜障害記録)。ファイル名と番号は雛形と同じにする
  - `adr/` — アーキテクチャ決定記録(1判断 = 1ファイル、`NNNN-slug.md`)
  - `poc/` — 技術検証の記録
  - `runbook/` — リリース手順・障害対応
