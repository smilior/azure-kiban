HOW IT WORKS ・ 開発はこの3手で回る

「◯◯を作りたい」と伝えると、企画書から実装まで専門エージェントのチームが起草します。エージェントは判断が要る場面で必ず停まり、あなたの承認を待ちます。あなたが書くのはチャットの指示だけです。

1題材を伝える。「◯◯を管理するシステムを作りたい」で企画が動き出す
2確認事項に答える。エージェントは業務ルールを勝手に決めず、質問リストで返してくる
3PRをマージする。マージ=承認記録。docs/ とコードが1段ずつ育つ
§1

はじめての方へ

読者は3種類です。自分の入口から入れば、全部を読む必要はありません。

FOR DRIVERS ・ 情シス/開発担当

運転する人

指示と検収で開発を進める。乗車券(5つの承認)を揃え、1ストーリーずつ回す。

実践ガイド ⓪ 立ち上げ編から → 作業中は ①流れ編・②手順編 を横に置く FOR PARTNERS ・ 利用部門

協力する人

業務の質問に答え、画面案を見て、できあがりを確かめる。

協力ガイドへ → 出番は3回だけ FOR APPROVERS ・ 上長

承認する人

承認は4種類。種類ごとに見るポイントが決まっている。

承認ガイドへ → 1件5〜10分
