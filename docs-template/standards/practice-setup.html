DRIVER'S GUIDE ⓪ ・ STD-AIDEV-008

実践ガイド ⓪ 立ち上げ編
─ 乗車券を手に入れるまで

案件で1回だけ通る、複製から設計承認までの道。①②とは逆に、ここは赤い駅ばかりです。あなたの承認が5つ揃うと乗車券になります。

0-1

リポジトリを複製する

①②のプロンプトはすべてチャットで打ちますが、最初のこれだけはターミナルです。テンプレートを複製すると、エージェント10体・雛形・規約が入った状態で、履歴のないまっさらなリポジトリが手に入ります。

gh repo create smilior/<案件名> --template smilior/ai-dev-template --private --clone
cd <案件名> && claude   # →「◯◯を作りたい。企画書から始めて」

複製した先での調整は3つだけです:

  • docs/standards/ は削除してよい。正本はテンプレート側にあり、閲覧はこの文書サイトでできます。残すと改訂が追いかけてこない古いコピーになります
  • .github/workflows/pages.yml を削除する。案件リポジトリは private のため Pages は動きません
  • CLAUDE.md の「案件確定時に記入」欄は、スタック決定後に tech-lead に任せます(駅5で埋まります)
0-2

路線図: 5つの承認駅

立ち上げの路線は、①とは配色が逆です。区間を走る(書類を起草する)のはエージェント、駅で停まって承認するのはあなた。5駅すべてで停車します。ここで決めたことが、①で列車が走る線路になります。

0-3

各駅で起きること

駅 1 ─ あなた × planner

企画GO: 効果を数字で見る

「備品管理システムを作りたい。企画書から始めて」で planner が動き、企画書(01_prd.md)が届きます。見るのは効果見込みが作業時間×頻度×人数の数字になっているか。前提が実態と違えば「週次じゃなく日次で引き直して」と返し、良ければ「GO。要件定義に進めて」で次の区間へ。上長のGOが要る案件は、この企画書を持って取りに行きます。

駅 2 ─ あなた × business-analyst

要件確定: 受け入れ条件を見る(最重要駅)

「この企画の要件定義をして」で要件定義書(02_requirements.md)と用語集が届きます。business-analyst は業務ルールを勝手に決めず「利用部門への確認事項」のリストを出してくるので、あなたが利用部門に聞いて回答を渡します。見るのは2つ: 受け入れ条件がGiven/When/Thenで書かれているか、ストーリーが2〜3日で終わるサイズか。ここで書いた受け入れ条件が、①でそのまま旗(テスト)になります。この駅の精度=開発全体の精度です。

駅 3 ─ あなた × ui-designer

画面合意: 利用部門と決める

「画面案を作って」で画面一覧・遷移図・決定メモ(05_screens.md)が届きます。あなた1人で決めず、利用部門に見せて合意を取ります(見せ方は協力ガイドにあります)。実装後の「思ってたのと違う」は、ほとんどがこの駅の通過漏れです。

駅 4 ─ あなた × architect

設計確定: 捨てた案の理由を見る

「技術選定をADRにまとめて」で構成図・ER図・ADR(04_architecture.md、adr/)が届きます。技術の中身に自信がなくても、見る点は2つで足ります: ADRに捨てた案とその理由が書かれているか、運用が1人で回る規模に収まっているか。この駅を出るとき、結合テストの契約(API仕様・スキーマ)が固まっています。

駅 5 ─ あなた × tech-lead

規約確定: 1画面かを見る

スタックが決まったら「CLAUDE.mdの案件欄を埋めて」で、tech-lead が命名規則・ディレクトリ構成・エラーハンドリング方針を記入します。見るのは分量だけ: 1画面に収まっているか。長い規約はAIが読み飛ばします。これで全駅通過です。

0-4

乗車券

5つの承認が、そのまま①流れ編に乗る条件です。書類の中身と完了条件の正本はフェーズ別成果物ガイドにあります。

ADMIT ONE
BOARDING PASS ・ 全5承認

乗車券 ─ ① 流れ編に進む条件

  • 企画書: 効果が数字(作業時間×頻度×人数)で示され、GOが出ている
  • 要件定義書: 受け入れ条件がGiven/When/Thenで書かれ、利用部門が確認事項に回答済み
  • 画面設計: 利用部門が画面案に合意している
  • 基本設計: API仕様とスキーマ(結合テストの契約)が決まっている
  • CLAUDE.md: 「案件確定時に記入」の欄が埋まっている

5つ揃ったら発車です。① 流れ編を開き、最初の一言は「US-01に着手して」。
