# 単体テスト観点

機能ごとの単体テスト観点を正常系・異常系・境界値で整理する。単体テストは詳細設計に対応（V字）。

## 観点整理の方針

- 各機能の処理・分岐・バリデーションを、正常系・異常系・境界値の3区分で洗い出す。
- 観点は「入力条件 → 期待結果」の形で検証可能に書く。受け入れ基準と対応させる。
- テストは `app/tests/features/<機能>/` に配置。参考は example 機能のテスト（`test_service.py` / `test_router.py`）。

## 網羅の目安

<!-- 機能ごとに、少なくとも下の区分を1件以上満たすこと。 -->

| 区分 | 最低限カバーする内容 |
|------|---------------------|
| 正常系 | 主要ユースケースの成功経路 |
| 異常系 | バリデーション違反・権限不足・対象不存在 |
| 境界値 | 桁数・件数・日付・数量の上下限とゼロ・空 |
