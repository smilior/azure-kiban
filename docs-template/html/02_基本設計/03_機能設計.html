機能設計書

{{アプリ名}} の各機能を外部から見た振る舞いとして定義する。

記入ガイド

機能ごとに「外部仕様」節を複製して記入する。内部の処理手順は書かない。

処理フロー・条件分岐・例外処理は 03_詳細設計/01_処理設計.md へ回す。

外部仕様のブロックは機能ごとに複製する。

受け入れ基準は Given-When-Then 形式で検証可能に書く。AI が完成を自己判定する基準になる。

（例）アイテム管理

参考実装は app/features/example/。

  • 目的: カテゴリに紐づくアイテムを管理し、有効/無効を切り替える
  • 対応画面: アイテム管理画面（/example/）
  • 権限: 参照は一般、作成・更新・削除は認証済みユーザー全員

入力項目

項目名 型 必須 制約・備考
アイテム名文字列必須1〜100文字
説明文字列任意制限なし
カテゴリIDUUID必須有効なカテゴリを指定
有効フラグ真偽任意既定 true

出力項目

項目名 型 備考
アイテム一覧配列ページネーション付き（PaginatedResponse）
カテゴリ名文字列カテゴリIDを結合して表示
作成日時 / 更新日時日時TimestampMixin により自動設定

受け入れ基準

  • 未ログイン状態でアイテム管理画面にアクセスすると、ログイン画面へリダイレクトされる
  • アイテム名を空で作成しようとすると、バリデーションエラーで登録されない
  • 一覧を検索キーワード付きで取得すると、アイテム名の部分一致のみ返る
  • アイテムを削除すると論理削除され、一覧に表示されない
