02 基本設計 · 06/07

API設計書

{{アプリ名}} の API 方針と一覧を定義する。

記入ガイド

API仕様の正は OpenAPI（FastAPI が /docs に自動生成）。本書は一覧と方針のみ扱う。

個別仕様が必要な場合のみ末尾の雛形を複製する。エラーコード体系は詳細設計へ。

OpenAPI で表現しきれない補足がある場合のみ、個別API仕様を複製して記入する。

共通規約

  • 実装構成: 機能ごとに app/features/<機能>/（router.py / service.py / schemas.py）。ルーティングは router、業務ロジックは service に置く。
  • 一覧レスポンス: 共通スキーマ PaginatedResponse[T]（items / total / page / per_page）を返す。
  • 削除: 論理削除。DELETE は deleted フラグを立て、物理削除しない。
  • 認証: 全エンドポイントで get_current_user_or_redirect により認証を確認する。画面は未認証時ログインへリダイレクト、API は 401 を返す。
  • 認可: ロールによる可否は 07_認可設計.md に従う。
  • エラーレスポンス: FastAPI 既定の {"detail": "..."} 形式に統一。主なコードは 400（バリデーション）/ 401（未認証）/ 403（権限なし）/ 404（未検出）/ 500（サーバーエラー）。
