権限チェックの実装方針

  • 認証は get_current_user_or_redirect（app/core/dependencies.py）で全エンドポイント共通に確認する。
  • 認可は FastAPI の依存性注入でチェックする。ロール判定をルーターの Depends に組み込み、権限不足時は 403 を返す。
  • チェックはサーバー側で必ず行う。画面側の表示制御は補助であり、権限判定の正としない。
  • 所有者チェック等の複雑な判定を追加する場合は 03_詳細設計/01_処理設計.md に記載し、本マトリクスと対応させる。参考実装は app/features/example/。
