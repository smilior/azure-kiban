# 監視・障害対応

{{アプリ名}} の監視項目・アラート・障害対応フロー。

## 監視項目

ヘルスチェックは `features/health` が提供する。`GET {{APP_ROOT_PATH}}/health` が JSON `{ "status": "ok"|"degraded", "checks": {...} }` を返す。`checks` は `app_db` / `external_db` / `redis` の疎通結果。

| 監視項目 | 対象 | 閾値 |
|---|---|---|
| (例) 死活監視 | `GET {{APP_ROOT_PATH}}/health` が `status: ok` | 連続 {{3}} 回失敗でアラート |
| DB 接続 | `checks.app_db` = ok | error 検知で即時 |
| Redis 接続 | `checks.redis`(セッションストア) = ok | error 検知で即時 |
| エラーレート | HTTP 5xx の割合 | {{1}} % 超で警告 |
| レイテンシ | 主要APIの応答時間 | {{500}} ms 超で警告 |
| リソース | CPU / メモリ / ディスク使用率 | {{80}} % 超で警告 |
