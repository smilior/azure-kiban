# 0000: ADRタイトル(名詞句で判断内容を書く)

ファイル名は連番4桁とスラッグを`-`でつなぐ(例: 0001-choose-database.md)。1判断につき1ファイルとする。

## 背景

判断が必要になった経緯を書く。制約(予算・期限・人員体制)と前提条件を含める。

## 選択肢

検討した選択肢を列挙する。各選択肢の特徴を1〜2行で書く。

- 選択肢A
- 選択肢B
- 選択肢C

## 決定

採用する選択肢を1文で書く。

## 理由

決定した理由を書く。運用体制・学習コスト・引き継ぎ可能性への影響を含める。

## 捨てた案とその理由

採用しなかった選択肢ごとに、捨てた理由を書く。

- 選択肢B: 理由
- 選択肢C: 理由

<details>
<summary>記入例: DBにPostgreSQLを選ぶ</summary>

## 背景

社内向け業務システムを1人体制で開発・運用する。データ量は月間数万件規模。

## 選択肢

- PostgreSQL
- MySQL
- DynamoDB

## 決定

PostgreSQLを採用する。

## 理由

マネージドサービス(RDS/Supabase等)が揃い、運用担当が自分1人でも保守できる。JSON型やウィンドウ関数など機能面の選択肢も広い。

## 捨てた案とその理由

- MySQL: 機能面で優位性がなく採用理由がない
- DynamoDB: スキーマレス設計への学習コストが高く、引き継ぎが困難

</details>
