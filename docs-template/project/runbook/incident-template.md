# 障害記録(雛形)

> 犯人探しではなく仕組みの改善を書く。

## 事象
> 何がいつ起きたかを事実だけで書く。推測は原因欄に書く。
- 発生日時: `<YYYY-MM-DD HH:MM>`〜`<YYYY-MM-DD HH:MM>`
- 検知経路: `<アラート名/通報者>`
- 症状: `<具体的な事象。例: API が 500 を返す>`

## 影響範囲
> 対象サービス・ユーザー数・影響時間を数字で書く。
- 対象サービス: `<service-name>`
- 影響ユーザー数/割合: `<件数または%>`
- 影響時間: `<分/時間>`
- データ損失の有無: `<あり/なし、ありの場合は範囲>`

## 時系列

| 時刻 | 出来事 | 対応者 |
| --- | --- | --- |
| `<HH:MM>` | 発生 | - |
| `<HH:MM>` | 検知 | `<氏名>` |
| `<HH:MM>` | 一次対応開始 | `<氏名>` |
| `<HH:MM>` | 復旧 | `<氏名>` |

## 原因
> 直接原因と、それを許した仕組み上の要因を分けて書く。
- 直接原因: `<何がトリガーになったか>`
- 背景要因: `<検知が遅れた理由・防止策が無かった理由など>`

## 再発防止
> 個人の注意喚起ではなく、仕組みの変更を書く。担当と期限を添える。
- 対応: `<変更内容>` / 担当: `<氏名>` / 期限: `<YYYY-MM-DD>`
- 対応: `<変更内容>` / 担当: `<氏名>` / 期限: `<YYYY-MM-DD>`
