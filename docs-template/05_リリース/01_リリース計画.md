## デプロイ手順

GitHub Actions の self-hosted runner が、GitHub Secrets / Variables から `.env.<ENV>` を生成してデプロイする。`develop` / `main` への push をトリガーに自動CDが走る。DEV はローカルで手動デプロイする。

1. 対象コードを `develop` にマージ(push)すると、自動CDで STG へデプロイされる
2. STG で動作確認する(ヘルスチェック `{{APP_ROOT_PATH}}/health` が `status: ok`、主要機能の疎通)
3. STG 確認後、`develop` を `main` にマージ(push)すると、自動CDで PRD へデプロイされる
4. PRD で動作確認する(手順2と同項目)

起動順序は データ基盤(PostgreSQL + Redis) → プロキシ基盤(nginx) → アプリ。逆順は `network shared_backend not found` で失敗する。

## ロールバック手順

1. 障害を検知したら、直前の正常バージョンのタグ / コミットを特定する
2. 安定版のコミットへ revert し、対象ブランチ(`main` / `develop`)へ push すると、自動CDで再デプロイされる
3. ヘルスチェックと主要機能で復旧を確認する
4. DB マイグレーションを含む場合、`alembic downgrade` の要否を判断する(データ破壊に注意)
