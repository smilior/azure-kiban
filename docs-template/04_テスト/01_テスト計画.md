# テスト計画

テストの範囲・V字対応・環境・合格基準・データ方針を定める。

## V字モデル対応

| 上流工程 | 対応する検証工程 | 合格基準の根拠 |
|----------|-----------------|---------------|
| 要件定義 | 受け入れテスト・総合テスト | 受け入れ基準・非機能要件 |
| 基本設計（外部） | 結合テスト | 機能設計書・API設計書 |
| 詳細設計（内部） | 単体テスト | 単体テスト観点表・コード |

## テスト範囲

| 区分 | 内容 |
|------|------|
| 対象 | {{アプリ名}} の全機能、認証・認可、API、画面表示、非機能要件（性能・セキュリティ） |
| 対象外 | 外部認証基盤（ap-safedx）内部、{{対象外とする範囲}} |

## 環境と実施時期

| テストレベル | 環境 | 実施時期 |
|-------------|------|----------|
| 単体テスト | CI（pytest） | 実装と同時・PR ごとに常時実行 |
| 結合テスト | STG | {{時期・マイルストーン}} |
| 総合テスト | STG | {{時期・マイルストーン}} |
| 受け入れテスト | STG（本番相当） | {{時期・マイルストーン}} |

## 合格基準

| テストレベル | 合格基準 |
|-------------|----------|
| 単体テスト | CI グリーン（全件パス）かつ カバレッジ {{数値}}% 以上 |
| 結合テスト | 全ケースが期待結果どおり。未達は起票し解消 |
| 総合テスト | 全シナリオ合格。性能・セキュリティは非機能要件の数値目標を満たす |
| 受け入れテスト | 全ユーザーシナリオ合格。ステークホルダー承認取得 |

- 性能・セキュリティの合格値は非機能要件定義書（`01_要件定義/`）の数値を参照する。本書に数値を重複記載しない。

## テストデータ方針

<!-- 本番の個人情報使用は禁止。ダミー生成またはマスキングを明記する。 -->

| 項目 | 方針 |
|------|------|
| 本番個人情報の使用 | 禁止。テストに実データの氏名・連絡先・アカウントを持ち込まない |
| データ生成 | ダミーデータ生成（単体は `app/tests/fixtures/factories.py`）またはマスキング済みデータ |
| 秘密情報 | `.env.<ENV>` を用い、テストログ・成果物に平文を残さない |
